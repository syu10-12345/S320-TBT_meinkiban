## 1. なぜ単ループPIDでは不十分なのか

### 現状の制御構造

```
              ┌──────────┐      ┌────────┐      ┌────────┐
  θ_target ──►│  PID     ├─ δe ►│ サーボ  ├─────►│ 機体   ├──► θ（ピッチ角）
  (= 0°)      │ (単ループ)│      │ KRS    │      │ 運動   │     │
              └──────────┘      └────────┘      └────────┘     │
                    ▲                                           │
                    │           BLE経由                         │
                    └──────────────────────────────────────────┘
```

今の soujyuukan_main は、「ピッチ角の誤差 → PID → エレベーター舵角」の1本道。
一見動きそうだが、航空機の運動にはこの構造が合わない理由が3つある。

### 理由1: 機体の応答が「2重の積分」を含む

エレベーターを動かしたとき、ピッチ角 θ が変わるまでの物理過程を追う:

```
δe（舵角）→ 尾翼に揚力が生じる → ピッチングモーメント M が生じる
→ 角加速度 θ̈ = M / Iyy → 角速度 θ̇ が変わる（1回目の積分）
→ ピッチ角 θ が変わる（2回目の積分）
```

つまり、δe から θ までの間に **積分が2回** 入っている。
制御工学では「2次系 (second-order system)」と呼ぶ。

PID制御器のI項もまた積分なので、単ループPIDでは系全体で
**積分が3つ**（機体の2つ + PIDのI項）重なる。
積分が多いほど位相が遅れ、系は不安定になりやすい。

> **直感的な理解**: 舵を切っても角度はすぐには変わらず、
> まず角速度が変わり、それが蓄積して角度になる。
> PIDが「角度がまだ変わらない！」とさらに舵を切ると、
> 既に角速度は十分ついているので、行き過ぎる（オーバーシュート）。

### 理由2: 外乱の伝搬経路

突風を受けたとき、まず **角速度 θ̇** が瞬間的に変化する。
その後、時間経過とともに **角度 θ** が変わる。

単ループPIDは θ しか見ていないので、θ̇ の変化を検知するのが遅い。
θ が十分変わってから初めて反応する＝常に手遅れ。

### 理由3: 速度依存性（後述の V² 問題）

## 2. 航空機のピッチ運動の物理

react-physics-lab のシミュレータで使っている運動方程式を整理する。

### ピッチの運動方程式

```
Iyy * θ̈ = M_main + M_tail
```

各項を展開すると:

```
M_tail = -l_tail * (1/2) * ρ * V² * S_tail * C_L_tail

C_L_tail = a_tail * (θ + δe + arctan(θ̇ * l_tail / V))
         ≈ a_tail * (θ + δe + θ̇ * l_tail / V)   （小角近似）

M_main = -(1/2) * ρ * V² * S_main * (C_L_main * dure - barc * Cm0)
```

ここで:
- `a_tail` = 尾翼の揚力傾斜 (= elek = 4.54 /rad)
- `l_tail` = 重心から尾翼までの距離 (= 5.79 m)
- `S_tail` = 尾翼面積 (= 3.22 m²)
- `dure` = 重心と空力中心のずれ (= -0.134 m)
- `V` = 対気速度

### 線形化した状態方程式

小角近似し、θ と θ̇ について線形化すると:

```
θ̈ = A * θ + B * θ̇ + C * δe + D
```

ここで（主要項のみ、定数は省略して構造を示す）:

```
A = -(ρV²/2Iyy) * [S_main * cl_α * dure + S_tail * a_tail * l_tail]

B = -(ρV/2Iyy) * S_tail * a_tail * l_tail²    ← 空力ダンピング

C = -(ρV²/2Iyy) * S_tail * a_tail * l_tail     ← 制御入力の効き

D = (ρV²/2Iyy) * S_main * barc * Cm0 + ...     ← 定常項
```

**重要な点: A, B, C すべてに V または V² が掛かっている。**

## 3. V²問題 ── 速度が変わるとゲインが変わる

### 何が起きるか

上の式で、制御入力の効き `C` に注目する:

```
C = -(ρV²/2Iyy) * S_tail * a_tail * l_tail
```

具体的な数値で計算してみる（react-physics-lab のパラメータ）:

```
V = 6 m/s のとき:
  C = -(1.225 * 36 / (2 * 197)) * 3.22 * 4.54 * 5.79
  C ≈ -1.225 * 36 * 3.22 * 4.54 * 5.79 / 394
  C ≈ -2393 / 394 ≈ -6.1 [rad/s² per rad]

V = 12 m/s のとき:
  C = -(1.225 * 144 / (2 * 197)) * 3.22 * 4.54 * 5.79
  C ≈ -9572 / 394 ≈ -24.3 [rad/s² per rad]
```

**速度が2倍になると、同じ舵角に対する角加速度が4倍になる。**

### PIDゲインへの影響

V = 6 m/s で良い感じにチューニングした Kp を使って V = 12 m/s で飛ぶと、
エレベーターの効きが4倍になるので、Kp が実質4倍効いているのと同じ。

```
低速 (V=6): Kp=1.0 → ちょうどいい応答
高速 (V=12): Kp=1.0 × 4倍の効き → 振動・発散の危険
```

逆に高速でチューニングすると低速で舵が足りなくなる。

```
 ゲイン
  │       ★安定に動作する範囲
  │      ┌─────┐
  │  ----│     │----  ← V=6 のとき
  │      └─────┘
  │  ★が狭い！
  │   ┌──┐
  │  -│  │----------  ← V=12 のとき（同じKpでもゲインが4倍）
  │   └──┘
  └────────────────── Kp
```

これが **「1組のPIDゲインでは全速度域をカバーできない」** 理由。

### 空力ダンピングも変わる

ダンピング項 B は V に比例する（V²ではない）:

```
B = -(ρV/2Iyy) * S_tail * a_tail * l_tail²
```

高速では自然なダンピングが強くなるので安定しやすく、
低速ではダンピングが弱くなるので不安定になりやすい。

つまり:
- **低速**: 舵が効かない + ダンピングが弱い → Kpを大きくしたいが振動しやすい
- **高速**: 舵が効きすぎ + ダンピングが強い → Kpを小さくしないと振動

この相反する要求を1組のゲインで満たすのは原理的に無理。

## 4. カスケード制御とは何か

### 基本アイデア

制御器を **2重のループ** にする。

```
                    外側ループ                  内側ループ
              ┌──────────────┐           ┌──────────────┐
  θ_target ──►│ PID_outer    ├─ θ̇_target►│ PID_inner    ├─ δe ──► 機体
              │ (ピッチ角)    │           │ (ピッチレート) │          │
              └──────┬───────┘           └──────┬───────┘          │
                     │    ▲                     │    ▲              │
                     │    │ θ                   │    │ θ̇            │
                     │    └─────────────────────┼────┼──────────────┘
                     │                          │    │
                     │                          │    └──── IMU ジャイロ
                     │                          │
                     └──────────────────────────┘
```

### なぜこれが良いのか

#### (a) 内側ループが「2重積分」を「1重積分」に変える

単ループ: δe → θ̈ → θ̇ → θ （積分2回を超えてθを制御）
カスケード: 内側は δe → θ̈ → θ̇ のみ（積分1回だけ）
            外側は θ̇_target → θ  （もう積分1回だけ）

**それぞれのループが1重積分だけを相手にする** ので、
各PIDの設計が格段に簡単になる。

#### (b) 内側ループが外乱を素早く打ち消す

突風で θ̇ が乱れた瞬間、内側ループがすぐに反応してδeを修正する。
外側ループが気づく頃には、内側ループが既にθ̇を制御下に戻している。

```
時間 →
         突風発生
            ↓
θ̇:  ────┐  ╱╲
         │ ╱  ╲──── 内側ループがすぐ修正
         │╱
θ:   ────────────── ほとんど変化しない

vs. 単ループ:

θ̇:  ────┐  ╱╲    ╱╲    ╱╲
         │ ╱  ╲  ╱  ╲  ╱  ╲── 誰も直接見ていないので振動
         │╱    ╲╱    ╲╱
θ:   ────┐
         │    ╱╲
         │   ╱  ╲────── θで初めて気づくので遅い
         │  ╱
```

#### (c) V²問題を内側ループが吸収する

内側ループ (θ̇制御) が正しく動いていれば、外側ループから見ると
「θ̇_target を指令したら、速度に関係なく θ̇ がそこに行く」ように見える。

V² の影響は内側ループが吸収し、外側ループはV²を気にしなくてよい。

```
外側ループの視点:
  「θ̇_target = 5°/s」と言ったら、
  V=6でもV=12でも θ̇ は 5°/s になる（内側ループのおかげ）

  → 外側のゲインは速度に依存しない（理想的には）
```

もちろん内側ループ自体は V² の影響を受けるので、
内側ループのゲインだけをスケジューリングすればよい。

## 5. 内側ループ（ピッチレート制御）の設計

### 目的

角速度 θ̇ を、指令値 θ̇_target に素早く追従させる。

### 制御則

内側ループは **P制御 または PD制御** で十分なことが多い。

```
δe = Kp_inner * (θ̇_target - θ̇)
```

なぜI項が不要か？

- 定常偏差は外側ループのI項が面倒を見る
- 内側ループにI項を入れると位相遅れが増え、高帯域が取れなくなる
- 内側ループは「速さ」が命。シンプルなほど良い

### 帯域の設計指針

内側ループの帯域 ω_inner は、外側ループの帯域 ω_outer の
**3〜10倍** が目安。

```
例:
  外側ループの応答: 1〜2秒で目標角に到達したい
    → ω_outer ≈ 1〜2 rad/s

  内側ループは:
    → ω_inner ≈ 5〜15 rad/s（ここでは制御周期30Hzの制約あり）
    → 30Hz = 188 rad/s なので、15 rad/s は余裕で実現可能
```

### 30Hzでの制御周期の制約

サンプリング定理により、制御帯域はサンプリング周波数の
1/6〜1/10 程度が実用的な上限:

```
30 Hz → 最大帯域 ≈ 3〜5 Hz ≈ 19〜31 rad/s
```

内側ループの帯域 15 rad/s は問題なく実現できる。

### θ̇ の取得方法

ICM20948 のジャイロスコープは角速度 θ̇ を直接出力する。
ピッチ角 θ から差分で微分するより遥かに高品質:

```
✗ θ̇ ≈ (θ[k] - θ[k-1]) / dt    ← ノイズ増幅、BLEのジッターで最悪
✓ θ̇ = ジャイロの直接出力        ← 低ノイズ、高帯域
```

これが meinkiban3 から θ だけでなく θ̇ も送るべき理由。

## 6. 外側ループ（ピッチ角制御）の設計

### 目的

ピッチ角 θ を目標角 θ_target に正確に追従させる。

### 制御則

外側ループは **PI制御** が基本:

```
θ̇_target = Kp_outer * (θ_target - θ) + Ki_outer * ∫(θ_target - θ) dt
```

- **P項**: 角度のずれに比例した角速度を指令
- **I項**: 定常偏差（トリムずれ、重心ずれ）を補償

D項は？ → 内側ループがθ̇を制御しているので、外側にD項は不要。
θ̇を直接制御しているのだから、「θの変化率を見て先読み」する必要がない。

### 直感的な理解

```
「今ピッチが目標より3°低い」
  → Kp_outer * 3° = 例えば 6°/s のピッチアップ角速度を指令
  → 内側ループがθ̇を6°/sに持っていく
  → θが上がり始める
  → 誤差が減り、θ̇_targetも自然に減速
  → θが目標に近づくとθ̇_target ≈ 0 → なめらかに収束
```

これは「角度が離れているほど速く回転し、近づくと減速する」
という、人間のパイロットと同じ操舵パターン。

### 帯域の設計

外側ループは内側ループより **十分遅く** する（1/3〜1/10）:

```
内側: ω_inner = 15 rad/s
外側: ω_outer = 2 rad/s （内側の 1/7.5）
```

これにより:
- 外側が「θ̇を5°/sにしろ」と言う
- 内側は十分速いので、外側から見ると「ほぼ即座にθ̇=5°/s」になる
- 外側は内側の動特性をほぼ無視でき、設計が独立する

## 7. ゲインスケジューリング

### カスケードでもスケジューリングが要る場面

内側ループが V² を吸収するが、V の変動幅が大きい場合は
内側ループのゲインを速度に応じて変える。

### 基本的な方法

制御入力の効き C は V² に比例するので、ゲインを V² で割る:

```
Kp_inner_actual = Kp_inner_base / (V/V_ref)²
```

ここで V_ref は基準速度（チューニングしたときの速度）。

```
V_ref = 8 m/s でチューニング: Kp_base = 2.0

V = 6 m/s のとき:
  Kp_actual = 2.0 / (6/8)² = 2.0 / 0.5625 = 3.56  （ゲイン増）

V = 12 m/s のとき:
  Kp_actual = 2.0 / (12/8)² = 2.0 / 2.25 = 0.89   （ゲイン減）
```

これにより、どの速度でも「PID出力 → 角加速度」の実効ゲインが一定になる。

### 実装方法

meinkiban3 には差圧センサー (SDP810) があり、対気速度 V を計算できる。
V を BLE で操縦桿に送り、PID ゲインを毎ループ補正する:

```c
// 擬似コード
float V = getAirspeedFromBLE();
float V_ratio_sq = (V / V_REF) * (V / V_REF);
float Kp_actual = Kp_base / V_ratio_sq;

float error_rate = theta_dot_target - theta_dot;
float delta_e = Kp_actual * error_rate;
```

### テーブル方式（代替）

計算が重い場合は、いくつかの速度帯でゲインをテーブルに持ち、
線形補間する方法もある:

```c
// ゲインテーブル
//            V=5    V=8    V=11   V=14  [m/s]
float Kp[] = {4.5,   2.0,   1.1,   0.65};
float Ki[] = {1.2,   0.5,   0.27,  0.16};

// 現在の V に対して線形補間
float Kp_now = interpolate(V_table, Kp, V_current);
```

## 8. このシステムへの適用

### 現在のハードウェア構成での制約

```
meinkiban3 (機体)                      soujyuukan_main (操縦桿)
┌─────────────────────┐               ┌──────────────────┐
│ ICM20948 → θ, θ̇    │──── BLE ────►│ PID → KRS サーボ  │
│ SDP810  → V        │  (遅延あり)    │                  │
│ GPS     → 位置     │               │ 可変抵抗→手動操舵  │
└─────────────────────┘               └──────────────────┘
```

BLE の遅延（20〜100+ms）は内側ループに致命的。
内側ループは高帯域が命なので、遅延は最小であるべき。

### 推奨構成: PIDを meinkiban3 側に移す

```
meinkiban3 (機体) ← PIDもここ
┌─────────────────────────────────────┐
│ ICM20948 → θ, θ̇（遅延ゼロ）          │
│ SDP810  → V（ゲインスケジュール用）    │
│                                     │
│ 内側ループ: θ̇_target → P制御 → δe   │  ← 高帯域、遅延なし
│ 外側ループ: θ_target → PI制御 → θ̇   │  ← 低帯域、遅延許容
│                                     │
│ δe → BLE送信                        │
└────────────────┬────────────────────┘
                 │ BLE（δe コマンド）
                 ▼
┌──────────────────────────┐
│ soujyuukan_main          │
│ 受信したδe → KRS サーボ   │ ← 単純なサーボドライバに徹する
│ 手動モード時は従来通り      │
└──────────────────────────┘
```

BLE で送るのが「ピッチ角（センサー値）」から「舵角（制御出力）」に変わる。
BLE の遅延はサーボコマンドの遅延になるが、これは内側ループの中ではなく
「内側ループの出力 → アクチュエータ」の経路なので、影響が小さい。

### 段階的な実装ステップ

#### Step 1: 現状の修正（まず動くようにする）

dtの型修正、vTaskDelete削除、unsigned double修正など。

#### Step 2: θ̇ の BLE 送信を追加

meinkiban3 の NavigationData に θ̇ (ジャイロのY軸) を追加:

```c
// meinkiban3 側
#pragma pack(push, 1)
struct NavigationData {
  float pitch;       // θ [°]
  float pitch_rate;  // θ̇ [°/s]  ← 追加
  float airspeed;    // V [m/s]  ← 追加（ゲインスケジュール用）
};
#pragma pack(pop)
```

#### Step 3: 操縦桿側でカスケードPIDを実装

まずは操縦桿側で試す（BLE遅延の影響はあるが、構造の検証ができる）:

```c
// 擬似コード
float theta_dot_target = pidCompute_outer(theta_target - currentPitch);
float delta_e = Kp_inner * (theta_dot_target - currentPitchRate);
```

#### Step 4: PIDを meinkiban3 に移動

検証が済んだら、PID ロジックを meinkiban3 に移植し、
BLE でδeコマンドを送る構成に変更。

#### Step 5: ゲインスケジューリングの追加

SDP810 から得た対気速度 V で内側ループのゲインを補正。

### react-physics-lab の改修

シミュレータもカスケード構造に対応させる:

```typescript
// 内側ループ（ピッチレート制御）
const error_rate = theta_dot_target - p.thetaDot * (180 / Math.PI);
p.now_u = Kp_inner * error_rate;

// 外側ループ（ピッチ角制御）
const error_angle = p.target - p.theta * (180 / Math.PI);
p.integral_outer += error_angle * dt;
const theta_dot_target = Kp_outer * error_angle + Ki_outer * p.integral_outer;
```

シミュレータでは θ̇（= p.thetaDot）が直接取得できるので、
カスケード制御の検証が容易。

## 9. 参考文献・次のステップ

### 学ぶべき制御理論

1. **ボード線図 (Bode Plot)**
   - ゲイン余裕・位相余裕の概念
   - なぜ積分が増えると位相が遅れるのか（積分1つ = -90°）
   - カスケードが位相余裕を改善する仕組み

2. **根軌跡 (Root Locus)**
   - ゲインを変えたときに極がどう動くか
   - V² でプラントゲインが変わると極がどう移動するか

3. **状態空間表現**
   - 航空機の運動を行列形式で書く: ẋ = Ax + Bu
